# 按运营主体拆表工具

将待拆表格按运营主体维度拆分，支持任意对接人配置。

## 快速开始

当用户说"拆主体表"、"主体拆表"、"zhutichaibiao"时：

```bash
cd /Users/bobobobo/Library/CloudStorage/Dropbox/Projects/p-LXX/.claude/skills/lx-zhutichaibiao/scripts
python split_by_zhuti.py
```

首次运行会自动进入交互式配置，按提示填写：
1. 项目根目录
2. 码表文件路径
3. 对接人代码
4. 工作目录名称

重新配置：`python split_by_zhuti.py --config`

## 工作流程

### 1. 首次配置

首次运行会进入交互式配置：
```
============================================================
按运营主体拆表工具 - 首次配置
============================================================
1. 项目根目录
   当前目录: /Users/xxx/projects
   回车使用当前目录，或输入路径:

2. 码表文件路径
   码表包含：城市、品牌、运营主体、对接人 的映射关系
   输入码表路径（相对或绝对路径）:

3. 对接人代码
   输入对接人代码（如 LWL）:

4. 工作目录
   目录下需包含：待拆表/、已拆表/、原表存档/ 三个子目录
   输入工作目录名称（默认: p-主体拆表）:
```

配置保存在 `assets/config.yaml`，重新配置运行 `--config` 参数。

### 2. 准备待拆文件

将待拆表格放入 `<工作目录>/待拆表/` 目录。

### 3. 运行拆分

```bash
python scripts/split_by_zhuti.py
```

### 4. 获取结果

- 拆分后的文件打包在 `<工作目录>/已拆表/` 目录
- 原始文件自动移入 `<工作目录>/原表存档/` 目录

## 输出格式

拆分后的表格统一应用以下格式：
- **字体**：等线 10号
- **对齐**：居中
- **列宽**：自适应内容
- **表头**：自动检测行数，保留原表头的合并单元格结构

## 配置文件

配置文件位于 `assets/config.yaml`，支持：

- **项目根目录**：工作目录的基准路径
- **码表路径**：城市-品牌-运营主体映射表
- **对接人**：筛选对应运营主体
- **工作目录**：包含待拆表/已拆表/原表存档
- **默认配置**：字段映射规则
- **特定配置**：按文件名关键词匹配的特殊规则

```yaml
# 用户配置
项目根目录: "/Users/xxx/projects/my-project"
码表路径: "/Users/xxx/data/码表.xlsx"
对接人: "LWL"
工作目录: "/Users/xxx/projects/my-project/p-主体拆表"

# 默认规则
默认:
  城市字段: [城市, 城市名称, 注册城市, 所属城市, city_name]
  品牌字段: [品牌, 品牌名称, 商家, 商家名称, 合作品牌, 合作商家]
  处理sheet: []

# 针对特定文件的配置
特定配置:
  - 匹配: "司机明细"
    城市字段: [注册城市]
    品牌字段: [品牌名称]
    处理sheet: [汇总1, 司机明细]
```

## 拆分规则

### 码表字段

| 字段 | 说明 |
|------|------|
| 城市 | 城市名称 |
| 品牌 | 品牌名称 |
| 新-运营主体 | 运营主体名称 |
| 对接人 | 对接人代码（按配置筛选） |

### 拆分逻辑

1. 读取配置文件，获取字段映射规则
2. 读取码表，筛选对接人的运营主体
3. 自动检测表头行数（支持多行表头）
4. 保留原表头的合并单元格结构
5. 根据品牌+城市匹配运营主体
6. 按运营主体拆分数据，一个运营主体一个文件
7. 应用统一样式（字体、对齐、列宽）
8. 打包为 zip 放入已拆表
9. 原表移入原表存档

## 注意事项

- 如果表缺少品牌或城市列，会提示错误跳过该表
- 如果没有匹配到对接人运营主体的数据，会提示
- 支持多 sheet 文件，每个 sheet 独立处理
- 可通过配置文件为不同文件设置不同规则
- 表头自动检测，支持多行表头和合并单元格
