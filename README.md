# 按运营主体拆表工具 (lx-zhutichaibiao)

将 Excel 表格按运营主体维度拆分，支持任意对接人配置，适用于网约车平台数据分析场景。

## 功能特点

- **交互式配置**：首次运行自动引导配置，无需手动编辑配置文件
- **灵活映射**：支持多字段名匹配（城市/城市名称/注册城市 等）
- **智能表头检测**：自动识别多行表头，保留合并单元格结构
- **特定文件规则**：按文件名关键词匹配，支持不同文件使用不同拆分规则
- **统一格式输出**：等线 10号、居中、自适应列宽

## 安装

### 方式一：通过 Claude Code 安装（推荐）

将以下指令发给 Claude Code：

```
从 https://github.com/yishudaxia/lx-zhutichaibiao 下载这个 skill，解压后放到 ~/.claude/skills/ 目录下，文件夹命名为 lx-zhutichaibiao
```

安装完成后，在 Claude Code 中说 "拆主体表"、"主体拆表"、"zhutichaibiao" 即可触发拆分流程。

### 方式二：手动安装

```bash
# 克隆仓库
git clone https://github.com/yishudaxia/lx-zhutichaibiao.git

# 复制到 Claude Code skills 目录
cp -r lx-zhutichaibiao ~/.claude/skills/
```

## 快速开始

### 1. 安装依赖

```bash
pip install -r scripts/requirements.txt
```

### 2. 运行拆分

```bash
python scripts/split_by_zhuti.py
```

首次运行会进入交互式配置：

```
============================================================
按运营主体拆表工具 - 首次配置
============================================================
1. 项目根目录
   当前目录: /Users/xxx/projects
   回车使用当前目录，或输入路径:

2. 码表文件路径
   码表包含：城市、品牌、运营主体、对接人 的映射关系
   输入码表路径（相对或绝对路径）:

3. 对接人选择
   需要拆出哪个对接人的表格？（多个用逗号分隔，回车拆出所有）:
   对接人:

4. 工作目录
   目录下需包含：待拆表/、已拆表/、原表存档/ 三个子目录
   输入工作目录名称（默认: p-主体拆表）:
```

### 3. 准备文件

将待拆表格放入 `<工作目录>/待拆表/` 目录。

### 4. 获取结果

- 拆分后的文件打包在 `<工作目录>/已拆表/`
- 原始文件自动移入 `<工作目录>/原表存档/`

## 码表要求

码表需包含以下字段：

| 字段 | 说明 |
|------|------|
| 城市 | 城市名称 |
| 品牌 | 品牌名称 |
| 新-运营主体 | 运营主体名称 |
| 对接人 | 对接人代码 |

## 配置文件

配置保存在 `assets/config.yaml`，支持：

```yaml
# 用户配置
项目根目录: "/Users/xxx/projects/my-project"
码表路径: "/Users/xxx/data/码表.xlsx"
对接人: "张三"
工作目录: "/Users/xxx/projects/my-project/p-主体拆表"

# 默认规则
默认:
  城市字段: [城市, 城市名称, 注册城市, 所属城市, city_name]
  品牌字段: [品牌, 品牌名称, 商家, 商家名称, 合作品牌, 合作商家]
  处理sheet: []  # 留空处理所有sheet

# 针对特定文件的配置
特定配置:
  - 匹配: "司机明细"           # 文件名包含此关键词时使用此规则
    城市字段: [注册城市]
    品牌字段: [品牌名称]
    处理sheet: [汇总1, 司机明细]  # 只处理指定sheet
```

### 重新配置

```bash
python scripts/split_by_zhuti.py --config
```

## 输出格式

拆分后的表格统一应用以下格式：
- **字体**：等线 10号
- **对齐**：居中
- **列宽**：自适应内容
- **表头**：自动检测行数，保留原表头的合并单元格结构

## 目录结构

```
lx-zhutichaibiao/
├── SKILL.md
├── README.md
├── assets/
│   ├── config.yaml          # 配置文件（首次运行自动生成）

├── 待拆表/                   # 待拆分的原始文件
├── 已拆表/                   # 拆分完成的zip包
└── 原表存档/                 # 已处理的原表存档
```

## 注意事项

- 如果表缺少品牌或城市列，会提示错误跳过该表
- 如果没有匹配到对接人运营主体的数据，会提示
- 支持多 sheet 文件，每个 sheet 独立处理
- 表头自动检测，支持多行表头和合并单元格
